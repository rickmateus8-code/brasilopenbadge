# Skill: DocMaster Cautious Engineering (Anti-Afobamento)

Esta skill estabelece os princípios de conduta ética e técnica para qualquer agente operando no ecossistema DocMaster.

## 🛑 PROTOCOLO DE CONDUTA (CONSTITUIÇÃO)
1. **MODO CIRÚRGICO:** Alterações devem ser diretas e pontuais. Jamais remova dados do próprio código ou exclua linhas sem revisão profunda.
2. **LIMITE DE DELEÇÃO:** É proibido remover mais de 10 linhas de código funcional sem autorização explícita do usuário.
3. **PLAN MODE OBRIGATÓRIO:** Todo ciclo de trabalho deve seguir rigorosamente: `Plan Mode` -> `Exibição do Plano` -> `Aprovação` -> `Execução`.
4. **ANT-AFOBAMENTO:** Proibido agir por impulso ou empolgação. O DocMaster é um sistema de alta complexidade; cada modificação exige a absorção total do fluxo e lógica antes de prosseguir.

## 🛡️ SEGURANÇA E ESTABILIDADE
1. **INTEGRIDADE VISUAL:** Proibido alterar layout, design, cores, fontes, alinhamento ou ordem de campos sem solicitação direta.
2. **BACKUP PREVENTIVO:** Realize sempre um backup do estado estável (ex: `.bak` ou `tar`) antes de alterações consideradas arriscadas.
3. **SINCRO-GITHUB:** Mantenha as alterações sincronizadas com o GitHub local para garantir pontos de restauração seguros. Deploy sempre via Wrangler.
4. **MEMÓRIA DE APRENDIZADO:** Todo erro de sincronia ou quebra de layout deve ser documentado na memória do projeto para evitar regressões.
